# CoffeeMan — работа с макетами (handoff → код)

Цель: агент берёт на себя максимум; вы **один раз** настраиваете доступ, дальше не «шлите скрины вручную» без необходимости.

## Что агент может сам

- **Встроенный браузер Cursor** (вкладка с Pixso): снимок структуры страницы, **скриншот видимой области** (и иногда full page), навигация по уже открытому файлу.
- **PDF** в `design/screenshots/CoffeeMan __ CLIENT.pdf` — опорный полный макет (лучше **облегчённая** копия для git — см. ниже).

## Ограничения (честно)

1. **Экспорт из Pixso «как в меню File → Export»** агент не делает в фоне: нужны клики по UI Pixso или ручной экспорт с вашей стороны **один раз**.
2. **Скриншоты из MCP** часто сохраняются в **локальную папку Cursor на вашей машине** (Temp), а не на удалённый сервер с репозиторием. Если проект открыт **удалённо**, положите копии скринов в `design/screenshots/` сами **или** работайте в workspace, где сохранение в проект поддерживается.
3. **Канвас Pixso** плохо попадает в «дерево доступности»: иногда в снимке видна только панель слоёв. Тогда нужен **выбранный фрейм на холсте** или экспорт фрейма в PNG из Pixso.

## Одноразовая настройка (сделайте один раз)

1. **Pixso в Cursor Browser**
   - Откройте файл **CoffeeMan __ CLIENT** во встроенном браузере.
   - Войдите в аккаунт **в этом же** окне (не только в Chrome).
   - Оставьте открытой страницу **Design** с нужной доской (например `design` / `UI-kit`).

2. **Артефакты в git**
   - Канонический путь к макетам: **`projects/coffeeman/design/screenshots/*.png`** (и zip/pdf рядом в этой же папке).
   - Файл **~400MB+** (`CoffeeMan __ CLIENT.pdf`) лучше **не коммитить**: храните рядом на диске / в облаке; для git достаточно PNG + при необходимости `CoffeeMan __ CLIENT.zip`.

3. **Именование скринов** (если экспортируете из Pixso сами, 2 минуты работы — потом 0 вопросов):
   - `01-home.png`, `02-catalog.png`, `03-product.png`, `04-cart-empty.png`, `05-checkout.png`, `06-blog.png`, `07-article.png`, `08-cafe.png`, `09-ui-kit-colors.png` …

## Как просить агента (копипаста)

> Сверстай каталог по `design/screenshots/catalog.png`, `design/screenshots/catalog item.png`, компоненты — `design/screenshots/header.png`, `design/screenshots/Chips.png`.

> Опирайся на `design/README.md` и PNG в `design/screenshots/`; PDF только если нужен полный обзор.

## Структура

```text
coffeeman/
  design/
    README.md                 ← этот файл
    screenshots/
      *.png                   ← экраны и UI-фрагменты
      CoffeeMan __ CLIENT.zip
      CoffeeMan __ CLIENT.pdf ← тяжёлый; в git лучше не класть
```

После настройки агент не должен дергать вас фразами «скачайте и пришлите» — достаточно поддерживать **актуальные файлы в `design/screenshots/`** и при необходимости открытую вкладку Pixso.
