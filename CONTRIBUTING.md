# Как участвовать

CoffeeMan — персональный проект, но идеи, баг-репорты и PR приветствуются.

## Если хочешь помочь

1. Сделай форк репозитория.
2. Создай ветку с понятным названием:
   ```bash
   git checkout -b feature/short-description
   ```
3. Внеси изменения. Старайся придерживаться **минимального diff** и **KISS**.
4. Убедись, что проект собирается:
   ```bash
   npm run lint
   npm run build
   ```
5. Если менял коллекции Payload, запусти:
   ```bash
   npm run generate:types
   npm run generate:importmap
   ```
6. Открой PR с кратким описанием: что, зачем, как проверял.

## Стиль

- TypeScript строго, без `any` без необходимости.
- Tailwind-классы по возможности; кастомные стили — через `src/styles/design-system.css` и CSS-переменные.
- Компоненты группируются по смыслу: `site/`, `ui/`, `catalog/`, `cart/`, `checkout/`, `blog/`.
- Не коммить `.env`, ключи, токены, `node_modules/`, `__pycache__/`.

## DoD

- `npm run lint` проходит без ошибок.
- `npm run build` собирается.
- Миграции применимы к PostgreSQL.
- UI соответствует вайбу продукта: тёплые тона, спокойный UX, контент-first.

Если сомневаешься — сначала обсуди в issue.
