# CoffeeMan — контракт для агента

**Версия:** 0.1.0 · **Вайб:** calm — «утренний кофе без суеты»

## 0. Режим работы

- **pair-programmer**, не автономный продакт. Пользователь задаёт приоритеты; агент исполняет, проверяет, сужает неопределённость.
- **Язык**: русский (продукт русскоязычный, e-commerce для РФ).
- **Тон**: `calm` — тёплый, спокойный, без суеты и крика.

## 1. Продукт

CoffeeMan — интернет-магазин кофе и чая с каталогом, корзиной, оформлением заказа, блогом и картой кофеен. Построен на Next.js 15 + Payload CMS 3 + PostgreSQL.

Ключевые ценности:

- **Визуальный комфорт**: бежевые тона, акцентный марсала, мягкие анимации.
- **Контент-first**: каталог, карточки товара, статьи и кофейны рассказывают историю продукта.
- **Простота поддержки**: Payload CMS даёт админку из коробки, Docker — единое окружение.

## 2. Вайб продукта

- **Тон**: `calm` — «утренний кофе без суеты».
- **3 правила**:
  1. Тёплая палитра (бежевый + марсала) и мягкие hover-эффекты — без резких контрастов и крика.
  2. Контент на первом месте: всё UI-решение служит товару, статье или заказу.
  3. Минимум шума: чистая типографика, продуманные отступы, анимации только там, где они помогают выбрать.

## 3. Стек и архитектура

| Область | Технология | Назначение |
|---------|------------|------------|
| Framework | Next.js 15 (App Router) | SSR/SSG витрины и админки |
| Runtime | React 19 | UI-компоненты |
| CMS / Admin | Payload CMS 3 | Headless CMS, админка, API |
| База данных | PostgreSQL 16 | Хранение товаров, заказов, контента |
| Стили | Tailwind CSS 4, `src/styles/design-system.css` | Атомарные + кастомные стили |
| Анимации | Framer Motion | Мягкие hover-эффекты и переходы |
| Состояние | Zustand | Локальный и персистентный стор корзины |
| Валидация | Zod | Формы оформления заказа |
| Почта | Nodemailer | Уведомления менеджеру |
| Контейнеры | Docker, docker compose | Единое окружение dev/prod |

## 4. Инженерная дисциплина

- **KISS / минимальный diff / YAGNI**. Не переписывай соседний код без причины.
- **Не ломай**: если не уверен на 100% — спроси, не делай предположений.
- **Не коммить** `.env`, `*.pem`, ключи, токены, пароли, `node_modules/`, `__pycache__/`.
- **Дизайн-решения** сверяй с `design/README.md`, `DESIGN_IMPLEMENTATION.md` и `ADVANCED_DESIGN_EFFECTS.md`.
- **Payload-коллекции**: после изменений запускай `npm run generate:types` и `npm run generate:importmap`.
- **Миграции**: изменения схемы БД должны быть в миграциях (`src/migrations/`).
- **TypeScript строго**, без `any` без необходимости.

## 5. Git workflow

- Коммить результат задачи — часть работы.
- Пушь, если пользователь просит или задача явно требует публикации.
- **Текущая ветка**: `main`.
- **Remotes**:
  - `origin` — GitHub (`git@github.com:Shugar86/coffeeman.git`)

## 6. Definition of Done

1. Код компилируется: `npm run build` проходит без ошибок.
2. Линтер чист: `npm run lint` без ошибок.
3. Миграции применимы к PostgreSQL: `npx payload migrate` отрабатывает.
4. Если менялась коллекция Payload — сгенерированы типы (`payload-types.ts`) и import map.
5. UI соответствует вайбу продукта: тёплые тона, спокойный UX, контент-first.
6. В отчёте: что изменилось, что запускал, риски и следующие шаги.

## 7. Эскалация

Спроси пользователя, если:

- затронуты секреты / `.env` / ключи / токены / платёжные провайдеры;
- перед тобой два равных архитектурных пути;
- задача касается prod-деплоя, домена, SSL или платёжной интеграции;
- требуется изменение публичного API или схемы данных, от которой зависят другие системы.
